Introduction

The bandit game is a command line game, the goal of which is to learn Linux commands. Through fun challenges, players are encouraged to learn by doing.

Motivation

Here I am going to share my solutions to the challenge, my intention is to present my way of solving the challenges because there may be several ways to do it in some cases. Please try to solve the challenge yourself before looking up answers, because that beats the purpose of the game.

Answers

The reading material, provided in the Helpful Reading Material section of each challange, is amply sufficient to complete the challenges and learn about the concepts we deal with. In the following section I will focus on the commands used only.

Level 0

SSH or Secure SHell is a secure connection protocol that allows users to connect to a remote server and execute commands on its command line.

The command used for that is: ssh bandit0@bandit.labs.overthewire.org -p 2220

The option -p 2220 here specifies which server port to connect to.

Level 0->1

The cat command is used to display the contents of a file. Here, we're instructed to view the contents of the readme file in the current directory.

This is done using the folowing command: cat readme

Level 1->2

cat ./-, by using cat ./-, we're explicitly specifying the file named - in the current directory. This circumvents any interpretation of - as a special character by preceding it with ./.

Level 2->3

cat 'spaces in this filename'. File names containing spaces can be tricky to handle in the command line. By enclosing the file name within single or double quotes, we ensure that the entire string is interpreted as a single argument.

Level 3->4

To find the hidden file: ls -a ./inhere. In Linux, file names beginning with a dot (.) are hidden by default. The ls -a command lists all files, including hidden ones.

Level 4->5

file ./inhere/* | grep 'ASCII text'The file command is used to determine the type of a file. By combining file with grep, we can filter the output to display only files of a certain type, such as ASCII text files, or: find ./inhere/* -readable .

Level 5->6

find ./inhere/* -readable -size 1033c -not -executable. The find command is a powerful tool for searching files based on various criteria. Here, we're tasked with finding files in the inhere directory that are readable but not executable and have a size of 1033c bytes.

Level 6->7

find / -type f -user bandit7 -group bandit6 2>/dev/null

The purpose of " 2>/dev/null " is to redirect stderr to a dump file.

Level 7->8

cat data.txt | grep 'millionth.'

Level 8->9

sort data.txt | uniq -u

Level 9->10

strings data.txt | grep -E "=+."

Level 10->11

Here we only need to decode the file: base64 -d data.txt

Level 11->12

Rot13: tr 'A-Za-z' 'N-ZA-Mn-za-m' <<< txt

Level 12->13

Commands used: mkdir, cp, mv, file, gunzip, bunzip2 and tar xvf

Level 13->14

ssh bandit14@localhost -i ssh.private -p 2220

Level 14->15

Commands used:

  • cat /etc/bandit_pass/bandit14
  • telnet localhost 30000

Level 15->16

openssl s_client -connect localhost:30001

Level 16->17

The command to find the port: nmap -T4 -A localhost -p 31000-32000

From the manpage manual: -A, to enable OS and version detection, script scanning, and traceroute; -T4 for faster execution; and then the hostname.

Then, after manually checking the output for each port, echo "previous level password" | openssl s_client -connect localhost:31790 -ign_eof

The server replied with an RSA private key, which I saved into a file in a directory I createdd in /tmp. I had to change the permissions of the file to 600 using the command chmod. I used the key file to login through ssh to bandit17@localhost using the -i parameter, as done previously.

Level 18->19

ssh bandit18@bandit.labs.overthewire.org -p 2220 cat readme

Level 23->24

To get the password for the next level, I did the following:

First, I created an empty file in /tmp and changed its permissions to 666 using chmod to allow all users to write into it.

Then, I created a bash script in the directory /var/spool/bandit24/foo that first displayed the content of the file /etc/bandit_pass/bandit24, using cat, and redirected the output to the file I created in the previous step. Additionally, I changed the permissions of the bash file to 777 so that all users can execute it.

Level 24->25

#!/bin/bash for i in {0000..9999}; do echo $(cat /etc/bandit_pass/bandit24) $i done | nc localhost 30002 > /tmp/output.txt

sort -u /tmp/output.txt
